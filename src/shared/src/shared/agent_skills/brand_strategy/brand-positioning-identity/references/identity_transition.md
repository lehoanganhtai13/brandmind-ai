# Identity Transition — Rebrand Only

## When to Use
Run for **refresh**, **repositioning**, and **full_rebrand** scopes.
Skip entirely for **new_brand** scope.

## Preserve-Discard Matrix

For each brand element, assign an action with rationale:

| Element | Current State | Action | Rationale | Equity Value |
|---------|--------------|--------|-----------|--------------|
| Brand name | Current name | preserve / evolve / replace / remove | Why this action | high / medium / low |
| Logo | Description | preserve / evolve / replace / remove | Why | high / medium / low |
| Color palette | Current colors | preserve / evolve / replace / remove | Why | high / medium / low |
| Typography | Current fonts | preserve / evolve / replace / remove | Why | high / medium / low |
| Tagline/slogan | Current tagline | preserve / evolve / replace / remove | Why | high / medium / low |
| Brand voice | Current tone | preserve / evolve / replace / remove | Why | high / medium / low |
| Visual style | Current imagery | preserve / evolve / replace / remove | Why | high / medium / low |
| Packaging | Current packaging | preserve / evolve / replace / remove | Why | high / medium / low |

### Action Definitions
- **Preserve**: Keep exactly as-is — high equity, strong recognition
- **Evolve**: Modernize or refine while maintaining recognizability
- **Replace**: New element needed — current version hurts or is irrelevant
- **Remove**: Element is no longer needed in the new identity system

## Summarize Transitions

After completing the matrix, group elements by action:
- **Elements preserved**: list all preserved items
- **Elements evolved**: list all evolved items
- **Elements replaced**: list all replaced items
- **Elements removed**: list all removed items

## Visual Bridge Strategy

Define how to transition visually so existing customers recognize the evolution:
- Gradual rollout plan (which elements change first)
- Transition period communication (explaining the change)
- Visual bridge elements that connect old to new identity

## DBA Continuity Plan

Ensure distinctive brand assets maintain continuity:
- Which DBAs carry over (even if evolved)?
- What new DBAs will be introduced?
- How will recognition be maintained during transition?
